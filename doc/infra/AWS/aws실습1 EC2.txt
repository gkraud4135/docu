1.https://aws.amazon.com/ko 주소이용 화면 우측상단 관리콘솔에
AWS 계정으로로그온합니다.

2.화면에 루트사용자가 선택되어 있는지 확인하고 화면표시된 루트사용자 전자메일주소를 이용 다음을 클릭하고 암호를 입력후 로그온합니다.

3.관리콘솔화면의 솔루션구축화면 아래 EC2를 사용하여 링크를 클릭합니다.

4.1단계 AMI 선택화면좌측하단에서 프리티어만 체크하고 처음에 표시된  Amazon Linux AMI 2018.03.0 (HVM), SSD Volume Type 을 선택합니다.
필요하다면 화면을 스크롤링하여 윈도우 서버같은 다른 운영체제 AMI를 선택할 수도 있습니다.

5.단계 2: 인스턴스 유형 선택화면에서 필터링 기준의 화살표를 선택하여 어떤 기준들이 있는지 확인해봅니다.
목록중에서 t2.micro를 선택(t2.micro의 인스턴스 유형은 1 개의 가상 CPU 및 메모리의 1 기가 사양입니다)하고 화면 하의 다음:인스턴스 세부구성을 단추를 클릭합니다.

6.단계 3: 인스턴스 세부 정보 구성에서 옵션들의드롭다운리스트를 통해 확인해보고 기본 구성값을 그대로 나둡니다
(특별히 종료방식은 중지와 종료가 있는데 중지는 그냥종료만하고 종료는 종료후 인스턴스를 삭제합니다)
확인 후 화면 하단 다음:스토리지 추가 단추를 클릭합니다.

7.단계 4: 스토리지 추가 화면에서 EC2인스턴스에 장착 할 스토리지를 설정합니다.Root장치는 꼭 있어야합니다.

볼륨유형:루트장치인지 추가장치인지를 설정하는 옵션입니다.기본적으로 루트장치는 EBS만 사용할 수 있고 추가 장치는 EBS와 인스턴스스토리지를 사용할 수 있습니다.
디바이스:Linux에서 표시되는 장치 이름입니다.
스냅샷:스냅샷생성시 ID입니다.
크기:스트리지크기이며 GB단위입니다.기본값 사용합니다.
볼륨유형:스토리지 볼륨유형입니다(SSD,마그네틱,프로비전된 IOPS를 선택할 수 있습니다.)
IOPS:볼륨유형을 프로비전된 IOPS를 선택시 값을 설정할 수 있습니다.
종료시 삭제:EC2인스턴스가 실행되고 있을때 스토리지가 실수로 삭제되는 것을 방지합니다 기본값 그대로 둡니다
설정이 완료되었으면 다음:태그추가를 클릭합니다.

8.단계 5: 태그 추가단계에서 EC2인스턴스의 태그를 설정합니다.key-value형식이며 다양한 내용을 추가 할 수 있습니다.태그별로 요금을 확인 할 수도 있습니다.키 입력상자에 Name ,값입력상자에 sample server라고 입력하고 다음:보안그룹구성 단추를 클릭합니다.

9.단계 6: 보안 그룹 구성 단계에서 방화벽인 Security Group을 설정합니다.
기본값 그대로 새 보안그룹 생성을 선택합니다.
Linux SSH 포트 22번이 기본적으로 설정된 보안그룹이 설정됩니다.
검토 및 시작 단추를 클릭합니다.

10.단계 7: 인스턴스 시작 검토 화면에서 이전 설정 값을 보거나 변경 할 수 있습니다. 변경할 사항이 없다면 시작하기 단추를 클릭합니다.

11.AWS에 처음 가입했고 EC2인스턴스에 접속을 위한 키 쌍을 생성하지 않았기 때문에 키쌍을 생성하라는 화면이 나옵니다. 새 키페어 생성을 선택하고 이름을 "awskeypair"라고 입력하고 키페어 다운로드 단추를 클릭합니다.pem파일이 다운로드가 완료되면 인스턴스 시작단추가 활성화 됩니다.

12.인스턴스 시작단추가 활성화되면 클릭하고 잠시 기다리면 EC2인스턴스가 생성되었다는 화면이 나오면 화면 하단 인스턴스보기 단추를 클릭합니다.
대시보드 화면이 표시되고 잠시기다리면 인스턴스상태가 초록색아이콘에 runing상태로 바뀝니다.이 상태가 되면 EC2 인스턴스를 사용 할 수 있습니다.
